# 🎟️ Event Management System - SQL Project

## 📌 Overview

This project is a MySQL-based Event Management System designed to manage events, venues, organizers, attendees, ticket bookings, and payments. It demonstrates database design, relationships, and advanced SQL operations for real-world scenarios.

---

## 🗂️ Database Structure

The system includes the following tables:

* **Events** – Stores event details
* **Venues** – Contains venue information
* **Organizers** – Stores organizer details
* **Attendees** – Holds attendee data
* **Tickets** – Manages ticket bookings
* **Payments** – Tracks payment transactions

Database and queries are available in:

* `event_management.sql` 

---

## ⚙️ Features

* Full database design with primary & foreign key relationships
* CRUD operations (Insert, Update, Delete, Select)
* Joins (INNER, LEFT, RIGHT)
* Subqueries and nested queries
* Aggregate functions (SUM, AVG, COUNT)
* Window functions (RANK, Running Total)
* CASE expressions for business logic
* Date & Time functions
* String manipulation functions

---

## 📈 Key Functionalities

* Event booking and ticket management
* Payment tracking and revenue calculation
* Attendee activity analysis
* Event demand categorization
* Top revenue generating events
* Pending and failed payment tracking

---

## 🛠️ Technologies Used

* MySQL
* SQL (DDL, DML, Advanced Queries)
